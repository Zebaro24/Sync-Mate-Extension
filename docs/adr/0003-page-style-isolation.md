# ADR-0003: Инжектируемые на страницу стили — только с префиксом `sync-mate-`

- **Статус:** Accepted
- **Дата:** 2026-06-27 (зафиксировано постфактум)

## Контекст

UI расширения на странице Rezka рисуется в чужом DOM. Короткие имена классов и `@keyframes`
(например `spin`) конфликтуют со стилями Rezka — Rezka может перетереть/переопределить их, и анимации
ломаются.

## Решение

Tailwind — только для popup. Всё, что инжектируется на страницу (`src/ui/*`), — vanilla DOM
(`element.style.*` или собственные `<style>`-теги), с уникальным префиксом `sync-mate-` для имён
классов и `@keyframes` (например `@keyframes sync-mate-spin` в `OverlayLoader`). См. [../styling.md](../styling.md).

## Последствия

- ➕ Изоляция от стилей Rezka; предсказуемый внешний вид.
- ➖ Нельзя пользоваться Tailwind на странице; стили пишутся вручную.
- 🔒 Enforced: `scripts/arch_lint_ext.py` требует префикс `sync-mate-` у всех литеральных `@keyframes`
  (гейт-проверка `arch`).
